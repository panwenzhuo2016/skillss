# 性能优化

分析代码性能问题并给出优化方案。

## 参数

$ARGUMENTS — 要优化的文件路径、方法名或性能问题描述（可选）

## 执行步骤

1. 确定优化目标（指定文件或 git diff 改动）
2. 读取并分析代码
3. 识别性能瓶颈
4. 给出优化方案并实施

## 检查项

### 后端
- **数据库**：N+1 查询、缺少索引、全表扫描、不必要的 JOIN
- **缓存**：频繁查询的数据是否加了缓存、缓存失效策略
- **并发**：锁的粒度、线程池配置、异步处理
- **内存**：大集合操作、流式处理、对象生命周期
- **IO**：批量操作替代逐条操作、连接池配置
- **算法**：时间复杂度、空间复杂度

### 前端
- **渲染**：不必要的重渲染、大列表虚拟化
- **网络**：请求合并、数据缓存、懒加载
- **包体积**：按需导入、代码分割、Tree Shaking
- **计算**：useMemo/useCallback 的合理使用、Web Worker

## 输出

按优先级排列优化建议：
1. 🔴 高优先级 — 明显影响用户体验
2. 🟡 中优先级 — 可观测到的性能提升
3. 🟢 低优先级 — 微优化，视情况处理
