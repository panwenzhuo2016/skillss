# 类型安全检查

检查 TypeScript/Java 代码的类型安全问题。

## 参数

$ARGUMENTS — 文件路径（可选，默认为 git diff 改动文件）

## 执行步骤

1. 确定目标文件
2. 分析类型使用情况
3. 输出类型安全报告

## TypeScript 检查项

### 严重问题
- `any` 类型的使用 — 替换为具体类型
- `as` 类型断言 — 是否可以用类型守卫替代
- `!` 非空断言 — 是否可以用可选链替代
- `@ts-ignore` / `@ts-expect-error` — 是否可以正确修复类型

### 建议改进
- 函数返回值是否有明确类型
- 接口/类型定义是否完整
- 泛型是否正确使用
- union 类型是否需要收窄
- 可选属性 `?` 是否在使用时做了判空

### API 类型
- API 请求参数和响应是否有类型定义
- 与后端接口是否匹配

## Java 检查项

- 泛型是否使用了原始类型（`List` vs `List<String>`）
- 可能的 ClassCastException
- Optional 是否正确使用
- `@Nullable` / `@NotNull` 注解是否完整

## 输出

列出所有类型问题，按严重程度排序，并提供修复代码。
