# 解决合并冲突

帮助解决 Git 合并冲突。

## 执行步骤

1. 执行 `git status` 查看冲突文件列表
2. 对每个冲突文件：
   - 读取文件内容，找到冲突标记（`<<<<<<<`、`=======`、`>>>>>>>`）
   - 分析双方的改动意图
   - 给出合并建议
3. 用户确认后解决冲突

## 分析方法

对每个冲突块：
1. **理解 HEAD 版本**（当前分支的改动）的意图
2. **理解传入版本**（合并分支的改动）的意图
3. 判断：
   - 两边改动不相关 → 保留双方
   - 两边改动同一功能 → 取更完整的版本
   - 两边改动有矛盾 → 分析哪个是正确的，或合并两者

## 输出

对每个冲突文件展示：
- 冲突位置和上下文
- 双方改动的说明
- 推荐的合并结果及理由

## 规则

- 每个冲突逐一展示，让用户逐个确认
- 不要默认选择某一方，要基于代码逻辑判断
- 如果不确定，展示所有可能的合并方案让用户选择
- 解决冲突后不自动 commit，让用户先验证
