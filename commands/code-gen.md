# 代码生成

根据描述生成完整的代码模块。

## 参数

$ARGUMENTS — 功能描述（如「用户管理的 CRUD」「文件上传接口」「权限校验拦截器」）

## 执行步骤

1. 理解功能需求
2. 查看项目现有代码的风格和模式
3. 生成符合项目规范的代码
4. 将代码写入正确的目录

## 生成原则

- **遵循项目现有模式**：查看类似功能的代码，保持一致
- **分层生成**：按项目的分层架构逐层生成
- **完整可用**：生成的代码应该可以直接运行

## 后端代码生成（按分层）

1. **Entity** — 数据库实体
2. **Mapper** — 数据访问层（MyBatis-Plus）
3. **Service / ServiceImpl** — 业务逻辑层
4. **Controller** — 控制层
5. **RO** — 请求对象（入参 DTO）
6. **VO** — 视图对象（出参 DTO）
7. **Enums** — 相关枚举

## 前端代码生成

1. **API 层** — 接口请求函数
2. **类型定义** — TypeScript 接口
3. **页面组件** — React 组件
4. **Hook** — 自定义 Hook（如果需要）

## 规则

- 生成前先展示代码结构，用户确认后再生成
- 使用项目已有的工具类和基础类（如 BaseEntity、BaseController）
- 参数校验使用项目统一的校验方式
- 不生成已存在的文件，如果需要修改已有文件则使用编辑
