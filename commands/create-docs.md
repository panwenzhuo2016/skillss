# 生成文档

为指定代码生成文档。

## 参数

$ARGUMENTS — 文件路径或模块名称

## 执行步骤

1. 读取目标代码文件
2. 分析代码结构（类、方法、接口）
3. 查看现有文档风格（如果有的话）
4. 生成符合项目风格的文档

## 文档类型

### API 文档（后端接口）
- 接口路径和方法
- 请求参数（类型、必填、说明）
- 响应结构
- 错误码
- 调用示例

### 类/模块文档
- 类的职责说明
- 公共方法列表及说明
- 使用示例
- 注意事项

### 代码注释
- 为公共方法添加 JavaDoc / JSDoc
- 为复杂逻辑添加行内注释
- 为类添加类级别说明

## 规则

- 文档语言与项目保持一致（中文项目用中文）
- 只记录「为什么」和「注意事项」，不重复代码已经表达的内容
- 保持简洁，不要写废话
- 如果项目有文档模板，遵循模板格式
