# 依赖检查

检查项目依赖的健康状态。

## 执行步骤

1. 读取依赖配置（package.json / build.gradle）
2. 分析依赖状态
3. 输出检查报告

## 检查项

### 未使用的依赖
- 扫描代码中的 import/require 语句
- 对比依赖列表，找出声明了但未使用的依赖
- 区分运行时依赖和开发依赖

### 重复依赖
- 功能重叠的包（如同时有 moment 和 dayjs）
- 同一个包的不同版本

### 安全风险
- 查看是否有已知漏洞的依赖版本
- 检查不再维护的包

### 版本状态
- 列出可以升级的依赖及最新版本
- 标注大版本升级（可能有 breaking changes）

## 输出格式

```
📦 依赖检查报告

✅ 健康依赖: X 个
⚠️ 可升级: X 个
❌ 未使用: X 个
🔴 安全风险: X 个

详细列表...
```

## 规则

- 建议删除未使用的依赖，但要确认确实没有被间接使用
- 升级建议优先考虑安全补丁版本
- 大版本升级需要评估 breaking changes
