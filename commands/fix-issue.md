# 修复 GitLab/GitHub Issue

根据 Issue 编号分析需求并实现修复。

## 参数

$ARGUMENTS — Issue 编号或 URL

## 执行步骤

1. **获取 Issue 信息**
   - 使用 `gh issue view <编号>` 获取 Issue 详情
   - 阅读标题、描述、评论、标签

2. **分析需求**
   - 理解 Issue 描述的问题或需求
   - 确定影响范围和优先级
   - 如果是 bug，分析复现步骤

3. **定位代码**
   - 根据 Issue 描述搜索相关代码
   - 理解相关模块的架构

4. **制定方案**
   - 列出修改方案
   - 评估影响范围
   - 向用户展示方案

5. **实施修改**
   - 按方案修改代码
   - 确保符合项目规范

6. **验证**
   - 运行相关测试
   - 确认修改不影响其他功能

## 规则

- 修改范围限定在 Issue 描述的范围内
- 如果 Issue 描述不清楚，列出疑问点让用户确认
- 复杂 Issue 建议拆分为多个子任务
- 修复完成后建议创建对应的测试用例
