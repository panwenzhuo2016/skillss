# 扫描 TODO/FIXME

扫描项目中的 TODO、FIXME、HACK、XXX 等标记。

## 参数

$ARGUMENTS — 扫描范围（文件路径或目录，可选，默认为整个项目）

## 执行步骤

1. 搜索代码中的标记注释：`TODO`、`FIXME`、`HACK`、`XXX`、`WARN`、`BUG`
2. 按类型和文件分类整理
3. 输出汇总报告

## 输出格式

### 统计
- TODO: X 个 | FIXME: X 个 | HACK: X 个 | 其他: X 个

### 详细列表

按优先级排序：
1. 🔴 **FIXME / BUG** — 需要修复的问题
2. 🟡 **HACK / XXX** — 临时方案，需要优化
3. 🟢 **TODO** — 待办事项

每条包含：
- 文件路径:行号
- 标记内容
- 相关代码上下文（前后 2 行）

### 建议
- 哪些可以直接解决
- 哪些已经过期可以删除
- 哪些需要创建 Issue 跟踪

## 规则

- 排除 node_modules、build、dist、.git 等目录
- 排除第三方库和生成的代码
- 如果 TODO 附带了作者或日期，也一并展示
