# 安全检查

对改动代码进行安全审计。

## 执行步骤

1. 执行 `git diff HEAD` 获取改动代码
2. 逐文件进行安全分析
3. 输出安全审计报告

## 检查项（OWASP Top 10）

### 注入攻击
- SQL 注入：是否使用参数化查询
- XSS：用户输入是否转义
- 命令注入：是否拼接系统命令
- LDAP/XML 注入

### 认证与授权
- 接口是否有权限校验
- 敏感操作是否验证身份
- 权限粒度是否足够
- Session/Token 管理

### 敏感数据
- 密码是否明文存储或传输
- 日志中是否打印敏感信息
- API 响应是否返回多余敏感字段
- 硬编码的密钥、密码、token

### 配置安全
- CORS 配置是否过于宽松
- 调试模式是否关闭
- 默认密码是否修改
- 依赖包是否有已知漏洞

### 其他
- 文件上传是否校验类型和大小
- 反序列化是否安全
- 重定向 URL 是否校验

## 输出格式

- 🔴 **高危** — 必须立即修复
- 🟡 **中危** — 建议尽快修复
- 🟢 **低危** — 建议关注

每条包含：风险描述、影响范围、修复建议、代码位置。
