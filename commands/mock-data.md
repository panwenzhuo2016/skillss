# 生成测试数据

为开发和测试生成模拟数据。

## 参数

$ARGUMENTS — 数据描述（如「10条用户数据」「订单列表JSON」「SQL INSERT」）

## 执行步骤

1. 理解需要什么数据
2. 读取相关的实体类/接口定义/表结构
3. 生成符合业务规则的测试数据

## 数据格式

根据需求生成不同格式：
- **JSON** — 前端 mock 数据、API 测试
- **SQL INSERT** — 数据库初始化、测试数据
- **TypeScript 对象** — 前端测试用例
- **Java 对象** — 后端测试用例
- **CSV/Excel** — 批量导入测试

## 数据质量

- 数据要**真实合理**（不要 test1、test2）
- 中文数据用真实的中文名、地址、公司名
- 日期要合理（不要未来日期或太久远的日期）
- 数值范围合理（价格、年龄、数量等）
- 枚举值覆盖所有可能的状态
- 关联数据保持引用完整性（外键关系）
- 包含边界情况（空字符串、最大值、特殊字符）

## 规则

- 不要包含真实的个人信息
- 手机号用虚拟号段（如 138xxxx）
- 邮箱用 example.com 域名
- 身份证号用虚拟号码
- 数据量可配置，默认 10 条
