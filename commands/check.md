# 代码质量检查

对项目或改动文件进行全面的代码质量检查。

## 执行步骤

1. 读取项目配置，识别项目类型和工具链
2. 执行 `git diff HEAD` 获取改动文件
3. 运行项目配置的检查工具
4. 汇总并输出检查报告

## 检查内容

### 自动化工具（如果项目配置了）
- **Lint 检查**：ESLint / Checkstyle / Spotless
- **类型检查**：TypeScript tsc / Java 编译
- **格式检查**：Prettier / Spotless
- **测试**：运行相关的单元测试

### 人工审查项
- **未使用的导入和变量**
- **TODO/FIXME 注释**是否需要处理
- **硬编码的配置值**是否应提取为配置
- **错误处理**是否完善
- **日志**关键操作是否有日志

## 输出格式

```
✅ 通过项  —  简要说明
❌ 未通过  —  问题描述 + 文件:行号 + 修复建议
⚠️ 警告    —  潜在问题 + 建议
```

如果发现可自动修复的问题，询问用户是否自动修复。
