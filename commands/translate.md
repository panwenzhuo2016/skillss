# 国际化翻译

为项目添加或更新国际化翻译。

## 参数

$ARGUMENTS — 要翻译的内容或语言文件路径（可选）

## 执行步骤

1. 识别项目的国际化方案（i18next、vue-i18n、Spring MessageSource 等）
2. 找到语言文件目录
3. 分析需要翻译的内容
4. 生成或更新翻译

## 场景

### 场景1：检查缺失翻译
- 扫描代码中的翻译 key（如 `t('xxx')`、`$t('xxx')`）
- 对比语言文件，找出缺失的翻译
- 补充缺失的翻译

### 场景2：新增翻译
- 为新功能的文案添加翻译 key 和对应的翻译文本
- 确保所有语言文件都同步更新

### 场景3：翻译文本
- 将指定的中文文案翻译为英文（或反向）
- 保持翻译风格与现有翻译一致

## 规则

- 翻译 key 使用语义化命名：`module.action.description`
- 不要翻译专有名词和品牌名
- 保持占位符格式一致（如 `{{name}}`、`{0}`）
- 翻译要自然通顺，不是逐字翻译
- 复数、性别等特殊语法要正确处理
