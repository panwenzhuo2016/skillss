# 代码重构

对指定代码进行重构，提升可读性和可维护性。

## 参数

$ARGUMENTS — 要重构的文件路径或模块名称（可选）

## 执行步骤

1. 如果指定了文件/模块，读取对应代码；否则读取 `git diff HEAD` 中的改动文件
2. 分析当前代码的问题点
3. 制定重构方案并展示给用户
4. 用户确认后执行重构

## 重构原则

- **不改变外部行为**：重构前后功能完全一致
- **小步前进**：每次只做一种重构，不要一次性改太多
- **保持测试通过**：如有测试，重构后必须通过

## 常见重构手法

- **提取方法**：过长的方法拆分为多个小方法
- **消除重复**：相似代码抽取为公共方法
- **简化条件**：复杂的 if-else 用策略模式、卫语句等简化
- **重命名**：让变量名、方法名更准确表达意图
- **移动代码**：将代码移到更合适的类或模块
- **减少参数**：过多参数封装为对象
- **消除魔法值**：硬编码常量提取为命名常量或枚举

## 输出

- 列出发现的问题和重构方案
- 执行重构并展示 diff
