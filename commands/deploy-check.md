# 部署前检查

在代码部署前执行全面的检查清单。

## 执行步骤

1. 读取项目配置，识别项目类型
2. 执行检查清单
3. 输出检查报告

## 检查清单

### 代码检查
- [ ] `git status` — 没有未提交的改动
- [ ] `git log` — 确认要部署的提交内容
- [ ] 没有调试代码（console.log、debugger、System.out.println）
- [ ] 没有硬编码的测试数据或测试 URL
- [ ] 没有注释掉的代码块

### 构建检查
- [ ] 构建成功（`npm run build` / `./gradlew build`）
- [ ] 没有编译警告或错误
- [ ] lint 检查通过

### 配置检查
- [ ] 环境变量配置正确
- [ ] 数据库连接指向正确环境
- [ ] API 地址指向正确环境
- [ ] 第三方服务配置（OSS、邮件、短信等）

### 数据库检查
- [ ] 迁移脚本是否已准备
- [ ] 是否有破坏性的 schema 变更
- [ ] 数据迁移脚本是否已验证

### 安全检查
- [ ] 敏感信息没有硬编码
- [ ] 调试接口已关闭
- [ ] CORS 配置正确

### 兼容性
- [ ] API 接口是否向后兼容
- [ ] 前后端版本是否匹配

## 输出

```
🚀 部署前检查报告

✅ 通过: X 项
❌ 未通过: X 项
⚠️ 需要确认: X 项

结论: 可以部署 / 建议修复后再部署
```
