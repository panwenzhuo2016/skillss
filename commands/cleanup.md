# 代码清理

清理项目中的无用代码和冗余内容。

## 参数

$ARGUMENTS — 文件路径或模块名称（可选，不填则扫描 git diff 改动文件）

## 执行步骤

1. 确定清理范围
2. 扫描并识别可清理项
3. 列出清理方案
4. 用户确认后执行清理

## 清理项

### 代码层面
- **未使用的导入**：import 了但没有使用的模块
- **未使用的变量和方法**：定义了但没被调用
- **注释掉的代码**：被注释的旧代码块
- **过期的 TODO/FIXME**：已完成但未删除的标记
- **空的代码块**：空的 catch、if、方法体
- **冗余代码**：无效的类型转换、多余的条件判断

### 文件层面
- **空文件**：没有实际内容的文件
- **重复文件**：内容相同或高度相似的文件
- **临时文件**：.bak、.tmp、.swp 等

### 依赖层面
- **未使用的依赖**：package.json 或 build.gradle 中声明但未使用的依赖
- **重复依赖**：功能重叠的依赖包

## 规则

- 每项清理都要说明理由
- 不确定是否在用的代码不要删
- 通过 git 搜索确认代码确实没有被引用
- 清理后确保编译/构建正常通过
