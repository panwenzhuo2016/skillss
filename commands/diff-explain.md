# 解释 Git Diff

用通俗易懂的语言解释当前的代码改动。

## 参数

$ARGUMENTS — 指定 commit hash 或分支名（可选，默认为 HEAD 的未提交改动）

## 执行步骤

1. 获取 diff 内容：
   - 无参数：`git diff HEAD`
   - 有 commit：`git diff <commit>..HEAD`
   - 有分支：`git diff <branch>...HEAD`
2. 逐文件分析改动
3. 生成结构化的改动说明

## 输出格式

### 改动概览
- 总共修改了 X 个文件，新增 X 行，删除 X 行
- 一句话总结本次改动的目的

### 逐文件说明
对每个文件：
- **文件路径** — 这个文件是做什么的
- **改动摘要** — 改了什么、为什么改
- **关键变化** — 列出重要的逻辑变更
- **潜在风险** — 如果有需要注意的地方

## 规则

- 用业务语言而非代码语言描述（「增加了用户注册时的邮箱校验」而非「在 validate 方法中添加了 email regex 匹配」）
- 区分核心改动和辅助改动（格式化、import 调整等）
- 标记破坏性变更（API 签名变化、数据库 schema 变更等）
