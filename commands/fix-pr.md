# 修复 PR 评审意见

获取 PR 的评审评论，并逐一修复。

## 参数

$ARGUMENTS — PR 编号或 URL

## 执行步骤

1. 使用 `gh pr view <PR编号>` 获取 PR 信息
2. 使用 `gh api repos/{owner}/{repo}/pulls/{number}/comments` 获取评审评论
3. 使用 `gh pr reviews <PR编号>` 获取审查意见
4. 逐条分析评审意见
5. 对每条意见：
   - 如果是代码修改建议 → 修改代码
   - 如果是疑问 → 分析后决定是否需要改动
   - 如果是风格建议 → 按项目规范修改
6. 修改完成后展示所有改动的摘要

## 规则

- 每修复一条意见，说明修改内容和原因
- 如果某条意见不同意，说明理由让用户决定
- 修复完成后不自动提交，让用户确认后再提交
- 保持修复的原子性，每条意见对应的修改尽量独立
