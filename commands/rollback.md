# 回滚方案

生成代码或数据库的回滚方案。

## 参数

$ARGUMENTS — 要回滚的内容描述（如「回滚最近3个提交」「回滚数据库迁移」）

## 执行步骤

1. 分析要回滚的内容
2. 评估回滚的影响
3. 生成回滚方案
4. 用户确认后执行

## 场景

### Git 代码回滚
- 查看提交历史：`git log --oneline -20`
- 分析要回滚的提交内容
- 生成回滚命令（优先用 `git revert`，保留历史）

### 数据库回滚
- 查看最近的迁移记录
- 生成反向迁移脚本
- 评估数据影响（是否会丢失数据）

### 配置回滚
- 对比配置变更
- 还原配置项

## 回滚策略

| 场景 | 推荐方式 | 原因 |
|------|----------|------|
| 已推送的提交 | `git revert` | 保留历史，安全 |
| 未推送的提交 | `git reset` | 可以清理历史 |
| 数据库新增字段 | 反向迁移脚本 | 可追踪 |
| 数据库删除字段 | 需要数据备份 | 不可逆 |

## 规则

- 默认使用最安全的回滚方式
- 回滚前先备份（代码 stash、数据库 dump）
- 评估回滚对其他功能的影响
- 数据库回滚尤其要谨慎，可能导致数据丢失
- 每一步操作前都要用户确认
