# 正则表达式助手

生成、解释或调试正则表达式。

## 参数

$ARGUMENTS — 需求描述或要解释的正则表达式

## 模式

### 模式1：生成正则
输入需求描述，生成对应的正则表达式。

示例输入：「匹配中国手机号」
输出：
```
/^1[3-9]\d{9}$/

解释：
- ^1      — 以 1 开头
- [3-9]   — 第二位是 3-9
- \d{9}   — 后面跟 9 位数字
- $       — 到此结束

匹配示例：13812345678 ✅
不匹配：12345678901 ❌
```

### 模式2：解释正则
输入正则表达式，用通俗语言逐部分解释。

### 模式3：调试正则
输入正则 + 测试字符串，分析为什么匹配/不匹配。

## 输出内容

1. 正则表达式本身
2. 逐部分解释
3. 匹配和不匹配的示例各 3 个
4. 适用语言的代码片段（Java / TypeScript）

## 规则

- 优先使用简单可读的正则，避免过度复杂
- 如果需求复杂，考虑拆分为多个简单正则
- 说明正则在不同语言中的差异（如果有）
- 考虑性能（避免灾难性回溯）
