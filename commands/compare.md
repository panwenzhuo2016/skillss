# 方案对比

对比多个技术方案或实现方式，帮助决策。

## 参数

$ARGUMENTS — 要对比的方案描述（如「Redis vs Caffeine 做缓存」「WebSocket vs SSE」）

## 执行步骤

1. 理解对比需求和使用场景
2. 分析每个方案的优劣
3. 结合项目实际情况给出推荐

## 对比维度

- **功能覆盖**：能否满足需求
- **性能**：延迟、吞吐量、资源消耗
- **复杂度**：实现难度、学习曲线、维护成本
- **可靠性**：稳定性、容错能力
- **扩展性**：是否容易扩展、是否有限制
- **生态**：社区活跃度、文档质量、周边工具
- **兼容性**：与项目现有技术栈的契合度

## 输出格式

### 对比表格

| 维度 | 方案A | 方案B |
|------|-------|-------|
| 功能 | ... | ... |
| 性能 | ... | ... |
| 复杂度 | ... | ... |

### 各方案详细分析
- 适用场景
- 优点
- 缺点
- 注意事项

### 推荐

基于项目实际情况（技术栈、团队能力、业务需求）给出推荐，并说明理由。

## 规则

- 客观公正，不偏向某个方案
- 结合项目的 CLAUDE.md 信息分析兼容性
- 如果都合适，说明各自最佳使用场景
